# 人生副本 / 玄学融合怪
# PRD Master v0.1

> 版本目标：把已有分散设计文档收束成一份可执行的总 PRD，作为后续原型、排期、实现和验证的统一依据。

> 当前验证口径：首个 launch target 以“可小范围试玩、可连续验证留存与竞争感的微信小游戏式 MVP”定义；应优先把现有主循环做厚、调稳，并补齐传播感与基础商业化承接，而不是继续横向开新系统。

---

# 1. 产品定义

## 1.1 产品一句话
**用生辰生成命格职业，然后通过抽卡、配装、刷秘境、打流年 Boss、冲同命理排行榜，完成一个具备微信小游戏商业化潜力的玄学融合怪。**

## 1.2 产品本质
这不是算命工具，也不是 RPG 看板，而是一个：

- **玄学命理驱动开局差异**
- **抽卡提供即时爽点**
- **装备与 build 提供中长期养成**
- **刷怪闯关提供稳定中循环**
- **排行榜提供传播与留存**

## 1.3 产品关键词
- 微信小游戏
- 玄学融合怪
- 抽卡
- 装备养成
- 刷怪闯关
- 自动战斗
- 命理分榜
- 商业化潜力

---

# 2. 为什么做

## 2.1 用户价值
玩家能获得：
- 低门槛进入：输入生辰即可拥有差异化身份
- 立即爽感：抽卡、掉装、战力上涨
- 长期目标：套装、Boss、榜单、赛季
- 社交话题：同命理对比、同日主冲榜、同日柱竞争

## 2.2 产品价值
这个方向兼具：
- 熟悉成熟玩法模型，不从 0 发明轮子
- 玄学题材天然有讨论度和传播性
- 可以承接微信小游戏的留存与付费逻辑
- 能做出显著差异化：**同命理排行榜**

---

# 3. 不做什么

明确排除：
- 纯八字报告工具
- 静态 RPG 面板网站
- 纯剧情文本选择游戏
- 纯 PVP 竞技产品
- 首阶段就做复杂实时联机

结论：
**优先做可持续运营的成长型小游戏，不做一次性 demo。**

# 4. 目标用户

## 4.1 核心用户
- 喜欢微信小游戏刷刷刷的人
- 对玄学 / 命理 / 运势有兴趣的人
- 喜欢抽卡、收集、配装、冲榜的人

## 4.2 扩展用户
- 喜欢轻中度养成产品的人
- 喜欢“我和别人有什么不同”这类身份型玩法的人
- 喜欢晒图、晒榜、晒 build 的社交玩家

## 4.3 用户心理抓手
- “测一下我是什么命格职业”
- “我今天运势怎么样，适合刷什么”
- “同样是水命，凭什么他比我强”
- “再抽一发就成型”
- “再刷一把 Boss 说不定出货”

# 5. 核心产品支柱

## 5.1 支柱一：命理做输入层
命理不是装饰，而是玩法入口：
- 决定职业起点
- 决定 build 适配
- 决定地图/Boss收益差异
- 决定排行榜切片维度

## 5.2 支柱二：抽卡做瞬时爽点
抽卡承担：
- 登录后的第一爽点
- 新手建立留存的第一动力
- 长线付费与追求的核心入口

## 5.3 支柱三：刷图做中循环
刷图承担：
- 出货期待
- 数值提升
- Build 验证
- 为 Boss 和榜单提供准备过程

## 5.4 支柱四：装备做长期成长
装备承担：
- 长线 build 深度
- 词条追求
- 套装追求
- 稀有追求

## 5.5 支柱五：排行榜做社交扩散
排行榜承担：
- 玩家竞争
- 日活拉回
- 分享传播
- 命理讨论度

---

# 6. 核心玩法闭环

## 6.1 外循环
**抽卡 → 强化 → 配装 → 冲榜**

## 6.2 中循环
**刷秘境 → 打 Boss → 掉装备 → 升层 → 再刷**

## 6.3 内循环
**自动战斗 → 技能触发 → 克制生效 → 结算掉落**

## 6.4 页面闭环
**主城 → 抽卡 → 配装 → 刷图 → 结算 → 强化 → Boss / 榜单 → 回主城**

这是产品最核心的动作链。

# 7. 命理系统接入设计

## 7.1 生辰输入
用户输入：
- 年
- 月
- 日
- 时辰

## 7.2 算法输出
基于 BaZi 引擎输出：
- 日主五行
- 身强身弱
- 喜用神
- 忌神
- 十神结构
- 五行得分

## 7.3 游戏化映射
### 日主五行 → 职业
- 木：青龙游侠
- 火：朱雀战法
- 土：麒麟守护
- 金：白虎刺客
- 水：玄武谋士

### 身强身弱 → 承压模型
- 身强：高风险高收益
- 身弱：更吃减伤/回复/护盾
- 中和：均衡成长

### 喜用神 → 推荐装备/地图/技能方向
- 喜金：暴击/穿透
- 喜木：成长/恢复
- 喜水：控制/技能
- 喜火：爆发/连击
- 喜土：生存/反伤

### 忌神 → 风险来源
- 穿忌神装收益下降
- 忌神 Boss 难度更高
- 特定日常副本波动更大

# 8. 战斗系统

## 8.1 战斗定位
- 轻量自动战斗
- 少量主动大招操作
- 重点不在手速，在 build、克制和配装

## 8.2 战斗单位
### 我方
- 1 个主角色
- 1 个守护灵

### 敌方
- 小怪
- 精英
- Boss

## 8.3 地图结构（MVP）
- 3 波普通怪
- 1 波精英（可选）
- 1 个 Boss

## 8.4 战斗节奏
- 单次刷图：1~3 分钟
- 小怪战：8~20 秒
- Boss 战：20~60 秒

## 8.5 Boss 原型
- 太岁冲命
- 破财煞
- 桃花劫
- 七杀压顶
- 枭神夺食

Boss 必须有机制，不是单纯数值更大。

# 17. 商业化结构

## 17.1 核心付费点
- 抽卡
- 月卡
- 通行证
- 强化材料包

## 17.2 次级付费点
- Boss 挑战券
- 守护灵成长包
- 稀有边框
- 称号
- 命盘皮肤

## 17.3 商业化原则
- 免费玩家可玩、可成长、可上分榜
- 微氪玩家有高性价比通路
- 中氪玩家 build 成型更快
- 高氪追求效率、稀有度和身份感

### 红线
- 不卖直接毕业
- 不做强逼氪墙
- 不让高氪无脑碾压所有命理分榜

# 18. MVP 范围

## 18.1 第一版必须有
1. 生辰生成命格职业
2. 1 套基础 BaZi → 游戏映射
3. 轻量自动战斗
4. 至少 3 张基础地图
5. 命器池 + 神通池
6. 基础装备掉落与强化
7. 1 个主线 Boss + 1 个每日 Boss
8. 总榜 + 同日主榜 + 今日 Boss 榜
9. 主城/抽卡/配装/战斗的完整闭环

## 18.2 第一版不要做
- 实时 PvP
- 聊天/公会
- 超多职业分支
- 超大世界观剧情
- 复杂多人协作
- 重型 meta 经济

MVP 核心就一句话：
**先验证“玄学命格 + 抽卡刷装 + 命理分榜”这个闭环是不是成立。**

# 19. 成功指标（方向）

## 玩家体感指标
- 3 秒懂这是玄学刷宝游戏
- 30 秒进入爽点
- 3 分钟愿意继续刷
- 1 天后愿意回来

## 产品验证指标
优先验证：
1. 玩家会不会继续刷图
2. 玩家会不会为了出货抽卡
3. 玩家会不会因为榜单继续成长
4. 玩家会不会对命理分榜产生讨论与分享

首阶段不急着看复杂商业指标，先看：
**玩法闭环是否成立，命理是否真的变成玩法优势。**

# 21. 当前结论

这条产品线已经从“人生副本 demo”转成了更清晰的方向：

## 正确方向
**玄学命格驱动的微信小游戏融合怪**
- 抽卡
- 装备
- 刷图
- Boss
- 榜单
- 命理分榜

## 不是
- 算命报告页
- 静态展示站
- 纯叙事小网页

后续所有原型、设计、开发，都应围绕这套主 PRD 推进。
